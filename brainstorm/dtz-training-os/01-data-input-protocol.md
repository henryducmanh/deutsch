# 01 — Data Input Protocol

## Goal

Create a standardized ingestion system for:

- DTZ model tests
- BAMF/g.a.s.t. documents
- audio
- transcripts
- tutor notes
- writing samples
- speaking samples

The system should treat documents as structured knowledge, not random files.

---

# Root Structure

```text
DTZ/
├── RAW/
│   ├── OFFICIAL/
│   ├── MODEL_TESTS/
│   ├── AUDIO/
│   ├── TRANSCRIPTS/
│   ├── WRITING_SAMPLES/
│   └── SPEAKING_SAMPLES/
│
├── PROCESSED/
├── STUDENT/
└── SESSIONS/
```

---

# Rules

## 1 file = 1 source unit

Examples:

- one exam PDF
- one transcript
- one audio source
- one scoring rubric

Do not merge unrelated materials.

---

# Model Test Rule

1 test = 1 folder.

Example:

```text
MODEL_TESTS/
└── DTZ_001/
    ├── META.md
    ├── exam.pdf
    ├── answer_key.pdf
    ├── transcript.pdf
    ├── audio/
    └── notes/
```

---

# META.md Structure

```text
Title:
Source:
Format:
Contains:
Answer Key:
Transcript:
Audio:
Priority:
Status:
Notes:
```

---

# Naming Rules

## Good

```text
DTZ_001_exam.pdf
DTZ_001_answer_key.pdf
DTZ_001_audio_01.mp3
```

## Bad

```text
new.pdf
scan001.pdf
abc.pdf
```

---

# Google Docs Rule

Preferred:

- export to PDF or DOCX
- store local copy

If keeping Google Docs links:

```text
source_link.txt
```

with:

```text
Source:
https://docs.google.com/...
```

---

# Raw vs Processed

## RAW

Never modify source files.

## PROCESSED

Generated by Claude Code / Cowork.

Contains:
- extracted patterns
- chunks
- templates
- traps
- summaries
